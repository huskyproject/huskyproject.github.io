The goal of this project is to develop a complete freeware suite of fidonet applications, running under Linux and other multitasking/multiuser systems. The suite should be comparable to now existing (and partly commercial) OS/2 systems.
To keep the configuration simple and reliable, all applications should use one set of configuration-files.
proposal configuration-files

Husky
portable Fidosoft Project

Ziel dieses Projektes ist es, eine komplette Freeware-Suite aus Fidonet-applikationen unter Linux und anderen multitasking/multiuser Systemen zur Verfügung stellen. Das Ergebnis soll  vergleichbar zu jetzt existierenden (teilweise kommerziellen) OS/2-Systemen werden. 
 Um eine einfache und sichere Konfiguration zu erreichen, sollen alle Applikationen einheitlich auf  eine Struktur von Konfigurations-Dateien zugreifen.
Entwurf Konfigurationsdatei

In this list, means: CVS modul name and link to latest snapshot. You should understand that the latest snapshot archives are generated automatically, are untested, and can contain all sorts of bugs and problems. If you are a new user, you should not start with the snapshots, but you should follow the links in the packet description which lead to the author's Homepage and the latest stable versions.

fido-area and mailinglist
 The international developer und support echo for husky is called fidosoft.husky. It is routed through the WWB and gated to a mailinglist. If you can't get the echo, ask Matthias Tichy or Tobias Ernst for a binkp or a ISDN or analog feed to this echo, or you can subscribe to the mailinglist (which is gated with the echo) and write your articles to: daemon@hit.priv.at   (unsubscribe here).
There is also a FTN fileecho called fidosoft.husky. It carries release source archives of the husky project software, i.E. no latest files are hatched here, only stable releases.
